# BLIND TURN V2 프로덕션 체크리스트

```text
Web URL:
Server URL:
배포 commit:
테스트 날짜/시간(KST):
테스터/기기:
```

방 코드 전체와 reconnectToken은 공유 문서/스크린샷/로그에 남기지 않습니다.

## 배포

- [ ] Web `/api/health`, Server `/health`가 HTTP 200
- [ ] health 응답에 방 코드, 닉네임, 토큰, 손패/큐가 없음
- [ ] Web이 HTTPS Server에 연결하고 `연결됨` 표시
- [ ] 허용되지 않은 Origin 거부
- [ ] Server 단일 인스턴스 유지

## 로비/연결

- [ ] 2~6인 입장, 7번째 `ROOM_FULL`
- [ ] 캐릭터 미선택 준비 거부, 변경 시 준비 해제
- [ ] 전원 연결/준비 후 방장만 시작
- [ ] 방장 연결 해제 시 좌석 기준 위임
- [ ] 로비 이탈자는 30초, 전원 이탈 방은 10분 후 정리

## 정보 은닉

- [ ] 상대 손패 카드/instanceId/뽑기 순서가 room 뷰에 없음
- [ ] 확정 전 상대 큐 장수/카드/순서/대상이 없음
- [ ] 잠금 후 사용 장수만 공개
- [ ] 합/회피 주사위는 판정 전 공개되지 않음
- [ ] `PRIVATE_CARD_DRAWN`, reconnectToken, 전체 GameState가 공개 이벤트/로그/health에 없음
- [ ] V1의 speedRoll, hiddenTieRoll, actionOrder, facing/허점 필드가 없음

## 타이머/동시성

- [ ] 60초 미확정은 카드 0장으로 처리
- [ ] 마지막 확정과 timeout 경합에도 라운드 결과 1회
- [ ] 같은 단계 동시 피해와 사망 순서가 모든 클라이언트에서 동일
- [ ] 재생 완료 누락 시 45초 후 진행
- [ ] 보상/덱 정리 60초 timeout 시 서로 다른 보상 2장/필요 제거 수 자동 선택
- [ ] 연출 중 새 이벤트가 와도 큐 순서 유지

## 전투/보상

- [ ] 일반 공격, 방어 중 피격, 회피 성공/실패, 반격, 합
- [ ] 같은 단계 사망자의 현재 행동 실행 및 다음 단계 취소
- [ ] 결투가/광전사/수호자/전술가 패시브
- [ ] 3라운드마다 직업 2 + 공용 1 후보 중 정확히 2장 선택
- [ ] 덱 10→12→14→15 성장, 초과 시 필요한 기존 카드만 영구 제거
- [ ] 이번 성장 카드 제거 금지, 무덤과 영구 제거 분리, 제거 카드 재셔플 금지
- [ ] 같은 카드 최대 2장, 공격 카드 최소 1장

## 재접속/채팅

- [ ] 선택 중 새로고침: 본인 손패/큐/확정/기한 복원
- [ ] 재생 중 새로고침: pending playback 재생 후 최종 HP 일치
- [ ] 보상/덱 정리 중 새로고침: 본인 후보·현재 선택·확정·기한·제거 불가 카드 복원
- [ ] 채팅 같은 방 한정, 100자, 초당 2개, 최근 50개
- [ ] 서버 닉네임 사용과 사망자 전송 거부
- [ ] 서버 재배포 뒤 `ROOM_SESSION_EXPIRED`로 안전 복귀

## 화면

- [ ] 2인/6인 데스크톱에서 공격자·대상·카드·단계 식별
- [ ] 같은 단계 카드가 플레이어 수와 무관하게 1박자에 동시 공개
- [ ] HP가 타격 전 먼저 감소하지 않고 단계 마지막에 모든 대상이 동시에 변경
- [ ] 피해 숫자 28px 이상, HP bar와 Controller가 동일한 600ms 박자 사용
- [ ] 390px에서 모바일 공격 경로, 카드 가로 스크롤, 54px 행동 버튼
- [ ] `prefers-reduced-motion`에서 흔들림 없이 결과 확인 가능

## 종료

- [ ] SIGTERM 시 action/playback/reward/disconnect/cleanup 타이머 정리
- [ ] 판정 오류는 `GAME_ENGINE_FAILURE`로 종료되고 재경기 가능
- [ ] 재경기에서 캐릭터/준비/덱/HP 초기화, 채팅 기록 유지

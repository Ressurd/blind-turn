# BLIND TURN 프로덕션 실전 테스트 체크리스트

배포 URL과 테스트 시간을 먼저 기록합니다.

```text
Web URL:
Server URL:
배포 commit:
테스트 날짜/시간:
테스터/기기:
```

문제가 생기면 방 코드 전체나 reconnectToken을 공유 문서·스크린샷에 남기지 말고, 발생 시각·브라우저·연결 상태·턴 번호·화면의 오류 코드만 기록합니다.

## 배포와 상태 확인

- [ ] Web `/api/health`가 HTTP 200을 반환한다.
- [ ] Server `/health`가 HTTP 200과 안전한 집계만 반환한다.
- [ ] health 응답에 방 코드, 닉네임, reconnectToken, 행동이 없다.
- [ ] Web 화면이 `연결됨` 상태를 표시한다.
- [ ] 브라우저가 HTTPS Server 도메인으로 Socket.IO 연결한다.
- [ ] Server가 한 인스턴스로 실행 중이다.

## 기본 연결

- [ ] PC 일반 브라우저에서 방을 생성한다.
- [ ] 시크릿 브라우저에서 입장한다.
- [ ] 모바일 LTE/5G에서 입장한다.
- [ ] 서로 다른 네트워크에서 게임할 수 있다.
- [ ] 허용하지 않은 Origin에서 연결이 거부되고 Server 로그에 오류 코드가 남는다.

## 2~6인 게임

- [ ] 2인 게임을 종료하고 재경기한다.
- [ ] 3인 게임을 완료한다.
- [ ] 4인 게임을 완료한다.
- [ ] 5인 게임을 완료한다.
- [ ] 6인 게임을 완료한다.
- [ ] 7번째 입장이 `ROOM_FULL`로 거부된다.

## 연결 문제와 복귀

- [ ] 행동 선택 중 새로고침 후 자신의 속도·제출 여부·남은 시간이 복원된다.
- [ ] 전투 연출 중 새로고침 후 최신 서버 상태로 복귀한다.
- [ ] Wi-Fi를 껐다 켠 뒤 `재연결 중`에서 `연결됨`으로 돌아온다.
- [ ] 모바일 화면을 잠갔다가 복귀한다.
- [ ] 연결이 끊긴 동안 기존 게임 화면이 즉시 초기화되지 않는다.
- [ ] 한 플레이어가 끊긴 상태로 제한 시간이 끝나도 턴이 한 번만 진행된다.
- [ ] 서버 재시작 후 `ROOM_SESSION_EXPIRED` 안내가 표시된다.
- [ ] `게임 정보 초기화`로 시작 화면에 돌아가 새 방을 만들 수 있다.

## 동시성과 타이머

- [ ] 여러 플레이어가 거의 동시에 행동을 제출한다.
- [ ] 마지막 제출과 30초 제한 시간 종료가 겹쳐도 턴이 한 번만 판정된다.
- [ ] 전투 연출 완료 신호가 일부 누락되어도 8초 후 다음 턴으로 진행한다.
- [ ] 게임 종료 직전 플레이어 연결이 끊겨도 결과가 일관된다.
- [ ] 재경기 요청 중 새로고침해도 로비 상태를 복원한다.
- [ ] 모든 플레이어가 끊긴 방은 10분 뒤 삭제된다.

## 정보 은닉

- [ ] 다른 플레이어의 속도가 room 뷰와 네트워크 응답에 노출되지 않는다.
- [ ] 다른 플레이어의 행동과 대상이 턴 판정 전에 노출되지 않는다.
- [ ] reconnectToken이 다른 플레이어 이벤트나 health 응답에 없다.
- [ ] Server 로그에 reconnectToken, 전체 GameState, 비공개 속도, 제출 행동이 없다.
- [ ] 공개 턴 이벤트에 `SPEED_ROLLED`가 없다.

## 종료와 오류 복구

- [ ] Railway 재배포 시 Server가 SIGTERM 로그를 남기고 제한 시간 안에 종료한다.
- [ ] 종료 시 방의 행동·다음 턴·연결 해제·cleanup 타이머가 정리된다.
- [ ] 잘못된 payload가 Server 프로세스를 종료시키지 않고 공통 오류로 응답한다.
- [ ] 전투 판정 오류가 발생하면 게임이 오류 종료되고 방장이 재경기할 수 있다.
- [ ] 배포 전후 `/health`에 준비되지 않은 프로세스가 정상으로 보고되지 않는다.

# BLIND TURN V2 멀티플레이 테스트 가이드

## 준비

```bash
pnpm install
pnpm dev
```

Web은 `http://localhost:3000`, Server는 `http://localhost:4000`입니다. 서로 다른 플레이어는 localStorage가 겹치지 않도록 일반 창/시크릿 창/다른 브라우저 프로필을 사용합니다.

## 기본 2인 흐름

1. A가 방을 만들고 B가 6자리 코드로 입장합니다.
2. 서로 다른 캐릭터를 선택합니다. 캐릭터를 바꾸면 준비가 해제되는지 확인합니다.
3. 둘 다 준비하고 방장이 시작합니다.
4. 시작 전체 덱 10장과 손패 5장, 덱/무덤/제거됨 장수와 캐릭터 HP를 확인합니다.
5. 플레이어 카드를 누르고 카드를 선택하면 별도 단계 선택 없이 첫 번째 빈 행동 순서에 즉시 예약되는지 확인합니다.
6. 하단의 작은 `내 행동` 요약에서 예약이 1번부터 연속되는지, 중앙 순서 편집창에서 위/아래 이동·개별 취소·전체 취소가 가능한지 확인합니다.
7. A가 확정해도 B 화면에는 A의 카드 수·종류·순서·대상이 보이지 않고 확정 여부만 보이는지 확인합니다.
8. 모두 확정하면 사용 장수 공개 후 1~3단계가 같은 순서로 재생되는지 확인합니다.
9. 같은 단계 카드가 인원수와 무관하게 한꺼번에 공개되고, 단계 마지막에 모든 피해 숫자와 HP 감소가 동시에 보이는지 확인합니다.
10. 모든 장면이 600ms 박자의 배수로 이어지고 재생 종료 후 서버 최종 HP와 일치하는지 확인합니다.

## 전투 시나리오

- 속공/강공 일반 피격
- 방어로 총 공격 피해 5 감소
- 회피 성공과 HP 유지
- 첫 회피 실패의 고정 10 피해, 이후 같은 단계 공격 정상 적용
- 지정 반격의 공격 취소 및 양쪽 10/2.5 동시 피해
- 상호 공격 합에서 좌석 순서대로 첫 번째 룰렛과 결과, 두 번째 룰렛과 결과, 카드/결투가 보너스, 최종 비교, 동률 재굴림이 겹치지 않고 표시되는지 확인
- 같은 단계에서 죽는 플레이어의 행동 실행
- 사망자의 다음 단계 카드 취소
- 광전사 직접 공격 +1, 수호자 HP 35
- 전술가 시작 6장 중 5장 선택, 남은 1장의 덱 복귀와 회수/교체/선별 추가 선택

## 보상

1. 3라운드 판정까지 진행합니다.
2. 중앙의 어두운 오버레이에 직업 2 + 공용 1이 표시되는지 확인합니다.
3. 후보 3장 중 정확히 2장을 선택하고, 다시 클릭해 해제/교체할 수 있는지 확인합니다.
4. 2장을 고르기 전에는 `선택한 카드 2장 확정` 버튼이 비활성화되는지 확인합니다.
5. 성장 화면에서 `현재 덱 보기`, 완료 인원과 선택 후 대기 화면을 확인합니다.
6. 보상과 덱 정리 제한 시간이 각각 60초인지 확인합니다.
7. 덱이 10→12→14장으로 성장하는지 확인합니다.
8. 9라운드에는 임시 16장에서 1장, 12라운드 이후에는 임시 17장에서 2장을 제거하는지 확인합니다.
9. 이번 성장 카드가 비활성화되고, 기존 손패/덱/무덤 카드는 선택 가능하며 마지막 공격 카드 제거가 거부되는지 확인합니다.
10. 제거 카드가 별도 `제거됨` 목록으로 이동하고 무덤 재셔플에 포함되지 않는지 확인합니다.
11. 보상/제거 시간을 넘겼을 때 서버가 정확한 수량을 자동 선택하고 다음 라운드를 시작하는지 확인합니다.

## 카드 조회와 재셔플

1. 상단의 손패·덱·무덤·전체 덱·제거됨을 각각 눌러 상세 모달을 엽니다.
2. 남은 덱은 종류별 수량만 보이고 정확한 드로우 순서나 instanceId가 없는지 확인합니다.
3. 전체 덱 합계에 예약 카드가 중복 없이 포함되고 위치가 `예약`으로 표시되는지 확인합니다.
4. 상대 뷰에는 각 더미의 장수만 있고 카드 구성이 없는지 확인합니다.
5. 덱을 비운 뒤 드로우하여 `덱이 비었습니다 → 무덤 셔플 → 덱 복원 → 드로우` 순서와 숫자 변화를 확인합니다.
6. 영구 제거 카드는 재셔플 연출과 새 덱 수량에 포함되지 않는지 확인합니다.

## 60초와 재생 큐

1. 한 명만 확정하고 다른 사람은 기다립니다.
2. 60초 뒤 미확정자의 카드가 0장으로 잠기고 라운드가 한 번만 판정되는지 확인합니다.
3. 마지막 확정과 timeout을 거의 동시에 발생시켜 중복 결과가 없는지 확인합니다.
4. 연출 도중 다음 결과가 도착해도 순서가 섞이지 않는지 확인합니다.
5. 일시 정지, 1/1.5/2배에서 모든 연출의 비율이 함께 바뀌는지 확인합니다.
6. 건너뛰기 후 중간 HP/덱 상태가 남지 않고 서버 상태와 일치하는지 확인합니다.
7. 한 클라이언트가 완료 신호를 보내지 않아도 45초 뒤 서버가 안전하게 진행하는지 확인합니다.

## 채팅

- 로비와 전투 중 같은 방 두 화면에 동일 메시지가 보임
- 연속 공백이 하나로 정리됨
- 빈 메시지/101자/초당 세 번째 메시지가 거부됨
- 다른 방에는 전송되지 않음
- 메시지 payload로 닉네임이나 playerId를 위조할 수 없음
- 시스템 메시지와 플레이어 메시지가 다르게 표시됨
- 사망자는 입력이 비활성화되고 서버도 거부함
- 50개 초과 시 최근 50개만 재접속 복원

## 재접속

행동 선택, 전투 재생, 보상 선택 각각에서 새로고침합니다.

- 같은 playerId로 손패/큐/확정/기한/채팅 복원
- 전투 재생 중에는 `pendingRoundPlayback`으로 해당 라운드 재생 후 최종 HP 동기화
- 상대 비공개 정보는 복원 응답에도 없음
- 방장 연결 해제 시 연결된 최저 좌석에게 방장 이전
- 서버 재시작 뒤에는 `ROOM_SESSION_EXPIRED` 처리

## 6인/모바일

- 6명 카드가 390px에서 세로로 읽히고 핵심 글자가 14px 미만이 아닌지 확인
- 공격자→대상 모바일 대체 경로가 보이고 피해 숫자가 화면 밖으로 잘리지 않는지 확인
- 성장 카드 3장과 덱 정리/덱 조회 카드가 가로 스크롤되고 선택 카드가 중앙에서 강조되는지 확인
- 행동 확정 버튼이 하단에서 접근 가능하며 예약 요약이 전투 화면을 과도하게 가리지 않는지 확인
- 데스크톱 SVG 연결선이 카드 중앙을 잇고 resize 후 재계산되는지 확인

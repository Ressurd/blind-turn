# BLIND TURN 멀티플레이 테스트 가이드

## 로컬 실행

```bash
pnpm install
pnpm dev
```

웹은 `http://localhost:3000`, 서버는 `http://localhost:4000`에서 실행된다. 브라우저 탭 두 개는 localStorage를 공유하므로 서로 다른 플레이어 테스트에는 일반 창과 시크릿 창, 다른 브라우저 프로필, 또는 서로 다른 브라우저를 사용한다.

## 2~6명 연결

1. 첫 번째 브라우저에서 닉네임을 입력하고 방을 만든다.
2. 6자리 코드를 복사한다.
3. 추가 브라우저에서 서로 다른 닉네임과 코드로 입장한다.
4. 로비에서 좌석, 방장, 접속, 준비 상태가 모든 화면에 동일하게 표시되는지 확인한다.
5. 최대 6명까지 입장되고 일곱 번째 입장은 거부되는지 확인한다.

## 정상 게임 진행

1. 전원이 준비한다.
2. 방장이 게임을 시작한다.
3. 각 브라우저에서 자신의 속도만 보이는지 비교한다.
4. 서로 다른 행동과 대상을 선택한다.
5. 제출 후 내 행동은 보이지만 상대 행동은 `LOCKED`로만 표시되는지 확인한다.
6. 전원 제출 후 각 브라우저의 이벤트 순서와 최종 HP가 같은지 확인한다.
7. 한 브라우저에서 `연출 건너뛰기`를 눌러도 서버 결과와 다른 브라우저 진행에 영향이 없는지 확인한다.
8. 다음 턴에 속도와 제출 상태가 새로 배정되는지 확인한다.

## 행동 제한 시간

1. 한 플레이어만 행동을 제출한다.
2. 다른 플레이어는 30초 동안 제출하지 않는다.
3. 제한 시간이 0이 되면 턴이 한 번만 판정되는지 확인한다.
4. 이벤트 로그에서 미제출 플레이어가 `시간 초과(PASS)`로 처리되고 자동 방어 효과를 얻지 않는지 확인한다.

## 연결 해제 및 재접속

1. 게임 도중 한 브라우저를 새로고침한다.
2. 같은 플레이어로 자동 복구되고 자신의 속도, 제출 상태, 남은 제한 시간이 유지되는지 확인한다.
3. 게스트 연결을 끊고 다른 화면에서 `AWAY`가 표시되는지 확인한다.
4. 방장 연결을 끊으면 연결된 플레이어 중 좌석이 가장 낮은 사람이 방장이 되는지 확인한다.
5. 로비에 다른 접속자가 남아 있으면 연결이 끊긴 플레이어가 30초 후 정리되는지 확인한다.
6. 모든 브라우저를 닫은 방은 재접속 유예를 위해 10분간 유지된 뒤 삭제되는지 확인한다.
7. 서버 재시작 후 기존 세션 복구를 시도하면 `ROOM_SESSION_EXPIRED`와 `게임 정보 초기화` 버튼이 표시되는지 확인한다.

## 종료 및 재경기

1. 한 명만 생존하거나 모두 사망할 때까지 진행한다.
2. 승자 또는 무승부, 최종 HP, 총 턴 수가 모든 화면에 같은지 확인한다.
3. 방장이 `로비로 돌아가기`를 누른다.
4. 같은 좌석과 닉네임을 유지한 채 체력·준비·게임 상태가 초기화되는지 확인한다.
5. 다시 전원 준비 후 두 번째 게임을 시작한다.

## 자동 테스트

```bash
pnpm test
pnpm lint
pnpm typecheck
pnpm build
```

Socket.IO 통합 테스트는 실제 임시 포트에 서버를 열고 클라이언트 두 개가 방 입장, 게임 시작, 행동 제출, 동일 결과 수신을 수행한다.

## 알려진 제한사항

- 방과 세션은 한 서버 프로세스의 메모리에만 저장되므로 서버 재시작 시 사라진다.
- 서버 재시작 후 기존 reconnectToken으로는 복구할 수 없으며 새 방을 만들어야 한다.
- 관전, 채팅, 공개 방 검색, 매칭은 포함하지 않는다.
- 여러 서버 인스턴스를 사용하는 배포에는 Redis 기반 Socket.IO 어댑터와 공유 세션 저장소가 필요하다.
- 브라우저 localStorage를 지우면 기존 재접속 토큰을 복구할 수 없다.

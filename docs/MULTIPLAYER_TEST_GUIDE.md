# BLIND TURN 멀티플레이 테스트 가이드

## 준비

```bash
pnpm dev
```

일반 창과 시크릿 창에서 `http://localhost:3000`을 열고 2인 방을 만듭니다. 6인 확인은 브라우저 창을 추가하거나 자동화 fixture를 사용합니다.

## 단일 행동 턴

1. 캐릭터를 고르고 모두 준비한 뒤 게임을 시작합니다.
2. 모든 직업이 시작 손패 5장인지 확인합니다. 전술가에게 별도 시작 손패 선택 화면이 나오면 안 됩니다.
3. 플레이어 카드를 먼저 누르면 그 대상에 사용할 수 있는 카드만 활성화되는지 확인합니다.
4. 카드 한 장을 고르면 단일 행동 요약에 카드와 대상이 표시되는지 확인합니다.
5. 다른 카드를 고르면 이전 선택이 교체되는지 확인합니다.
6. 선택 취소와 PASS가 동작하고 PASS가 카드를 소비하지 않는지 확인합니다.
7. 확정 전 상대에게 카드/대상이 보이지 않고 확정 여부와 카드 장수만 보이는지 확인합니다.
8. 미확정 상태로 60초가 지나면 서버가 PASS로 처리하는지 확인합니다.

## 전투 판정

- 일반 공격과 방어 중 피격
- 회피 성공과 실패 고정 10 피해
- 지정 반격의 공격자 10 / 반격자 2.5 동시 피해
- 상호 공격 합과 동률 재굴림 룰렛
- 합 패자의 현재 공격만 실패하고 “다음 행동 취소”가 없는지
- 같은 턴에 사망한 플레이어가 확정한 행동도 실행되는지
- 2인과 6인 모두 선택 카드가 한 번에 공개되고 HP가 피해 전에 먼저 줄지 않는지

## 성장

3턴 판정을 완료합니다.

- 일반 직업: 서로 다른 직업 2 + 공용 1, 총 3장
- 전술가: 서로 다른 전술가 3 + 공용 1, 총 4장
- 두 직업 모두 정확히 2장만 확정 가능
- 전술가는 PC에서 4열 또는 2×2, 모바일에서 가로 스크롤
- 두 장 선택 시 나머지 카드에 “이번에 포기할 카드” 표시
- 다른 플레이어에게 후보/선택/포기 카드가 노출되지 않음
- 시간 초과 시 서버가 서로 다른 2장을 자동 선택

9턴에는 임시 16장에서 기존 카드 1장을 제거해 15장이 되는지 확인합니다.

## 재접속

각 상태에서 새로고침한 뒤 같은 정보가 복원되는지 확인합니다.

- 손패, 선택 행동/대상, 확정 여부, 남은 시간
- 전술가의 동일한 성장 후보 4장과 현재 선택 0~2장
- 덱 제거 선택과 남은 시간
- 재생 중인 전투 이벤트와 displayState 진행

후보를 다시 생성하거나 이미 확정한 선택을 변경할 수 있으면 실패입니다.

## 화면 크기

- 데스크톱 1366×768, 1920×1080: 전체 페이지 스크롤 없이 전장/손패/행동 요약 확인
- 모바일 390px: 대상과 공격 방향, 피해 숫자, 손패, PASS/확정 버튼이 겹치지 않음
- 6인: 각 카드에 닉네임, HP, 손패/덱/무덤/전체 장수, 행동 확정 여부가 식별됨

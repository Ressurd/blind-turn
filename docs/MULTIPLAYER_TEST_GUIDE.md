# BLIND TURN V2 멀티플레이 테스트 가이드

## 준비

```bash
pnpm install
pnpm dev
```

Web은 `http://localhost:3000`, Server는 `http://localhost:4000`입니다. 서로 다른 플레이어는 localStorage가 겹치지 않도록 일반 창/시크릿 창/다른 브라우저 프로필을 사용합니다.

## 기본 2인 흐름

1. A가 방을 만들고 B가 6자리 코드로 입장합니다.
2. 서로 다른 캐릭터를 선택합니다. 캐릭터를 바꾸면 준비가 해제되는지 확인합니다.
3. 둘 다 준비하고 방장이 시작합니다.
4. 시작 손패 3장, 손패/뽑기/버림 장수와 캐릭터 HP를 확인합니다.
5. 손패에서 0~3장을 예약하고 공격/반격 대상과 순서를 지정합니다.
6. A가 확정해도 B 화면에는 A의 카드 수·종류·순서·대상이 보이지 않고 확정 여부만 보이는지 확인합니다.
7. 모두 확정하면 사용 장수 공개 후 1~3단계가 같은 순서로 재생되는지 확인합니다.
8. 피해 숫자와 HP 감소가 이벤트 순서대로 보이고 재생 종료 후 서버 최종 HP와 일치하는지 확인합니다.

## 전투 시나리오

- 속공/강공 일반 피격
- 방어로 총 공격 피해 5 감소
- 회피 성공과 HP 유지
- 첫 회피 실패의 고정 10 피해, 이후 같은 단계 공격 정상 적용
- 지정 반격의 공격 취소 및 양쪽 10/2.5 동시 피해
- 상호 공격 합, 카드/결투가 보너스, 동률 재굴림
- 같은 단계에서 죽는 플레이어의 행동 실행
- 사망자의 다음 단계 카드 취소
- 광전사 직접 공격 +1, 수호자 HP 35
- 전술가 시작 4장 중 3장 선택과 회수/교체/선별 추가 선택

## 보상

1. 3라운드 판정까지 진행합니다.
2. 각 플레이어의 후보가 본인에게만 보이며 직업 2 + 공용 1인지 확인합니다.
3. 제한 시간이 30초인지 확인합니다.
4. 첫 선택 후 덱 8→9, 두 번째 보상 후 9→10인지 확인합니다.
5. 덱 10장에서 보상을 받으면 기존 카드 제거 화면이 나오는지 확인합니다.
6. 새 보상이 제거 후보에 없고, 마지막 공격 카드 제거가 거부되는지 확인합니다.
7. 보상/제거 시간을 넘겼을 때 서버가 선택하고 다음 라운드를 시작하는지 확인합니다.

## 60초와 재생 큐

1. 한 명만 확정하고 다른 사람은 기다립니다.
2. 60초 뒤 미확정자의 카드가 0장으로 잠기고 라운드가 한 번만 판정되는지 확인합니다.
3. 마지막 확정과 timeout을 거의 동시에 발생시켜 중복 결과가 없는지 확인합니다.
4. 연출 도중 다음 결과가 도착해도 순서가 섞이지 않는지 확인합니다.
5. 일시 정지, 1/1.5/2배, 건너뛰기를 확인합니다.
6. 한 클라이언트가 완료 신호를 보내지 않아도 8초 뒤 서버가 진행하는지 확인합니다.

## 채팅

- 로비와 전투 중 같은 방 두 화면에 동일 메시지가 보임
- 연속 공백이 하나로 정리됨
- 빈 메시지/101자/초당 세 번째 메시지가 거부됨
- 다른 방에는 전송되지 않음
- 메시지 payload로 닉네임이나 playerId를 위조할 수 없음
- 시스템 메시지와 플레이어 메시지가 다르게 표시됨
- 사망자는 입력이 비활성화되고 서버도 거부함
- 50개 초과 시 최근 50개만 재접속 복원

## 재접속

행동 선택, 전투 재생, 보상 선택 각각에서 새로고침합니다.

- 같은 playerId로 손패/큐/확정/기한/채팅 복원
- 전투 재생 중에는 `pendingRoundPlayback`으로 해당 라운드 재생 후 최종 HP 동기화
- 상대 비공개 정보는 복원 응답에도 없음
- 방장 연결 해제 시 연결된 최저 좌석에게 방장 이전
- 서버 재시작 뒤에는 `ROOM_SESSION_EXPIRED` 처리

## 6인/모바일

- 6명 카드가 390px에서 세로로 읽히고 핵심 글자가 14px 미만이 아닌지 확인
- 공격자→대상 모바일 대체 경로가 보이고 피해 숫자가 화면 밖으로 잘리지 않는지 확인
- 카드 손패는 가로 스크롤되고 54px 이상 확정 버튼이 전투 카드와 겹치지 않는지 확인
- 데스크톱 SVG 연결선이 카드 중앙을 잇고 resize 후 재계산되는지 확인
